##  Project Overview
This project presents an interactive **Restaurant Sales Analytics Dashboard** built using **Excel** to analyze sales performance, customer orders, and revenue trends.

The dashboard transforms raw transactional data into clear business insights through **data cleaning, feature engineering, KPIs, and visualizations**, making it a strong portfolio project for **Data Analyst roles**.

---

##  Dataset Information
- **Source:** Kaggle
- **Domain:** Restaurant / Food Sales
- **Size:** 15,777 rows × 9 columns
- **Data Type:** Transactional sales records

### Key Columns
- Item
- Category
- Quantity
- Order Total
- Payment Method
- Month Name
- Day Name
- Year

---

##  Data Cleaning
The following preprocessing steps were applied:

- Removed **Price column** due to missing values
- Handled missing values in Order Date
- Extracted **Day, Month, Year** from Order Date
- Standardized column formats

---

##  Feature Engineering
Created new calculated fields:

- **Custom Price = Order Total / Quantity**
- Month column
- Day column
- Year column

These features improved time-based and pricing analysis.

---

##  KPIs
Main business metrics included:

- Total Sales
- Total Orders
- Revenue by Category
- Revenue by Payment Method
- Monthly Revenue Trend
- Yearly Sales Performance

---

##  Dashboard Structure (Single Page)

### Visualizations
- Total Sales by Item (Bar Chart)
- Total Sales by Category
- Sales by Payment Method
- Sales by Month
- Sales by Year
- Interactive Slicers for filtering

 Helps track best-selling items, peak months, and customer payment behavior.
